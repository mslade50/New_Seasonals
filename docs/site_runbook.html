6. Trade charts (the Charts tab) — how to update

Added 2026-06-15. Per-trade candlestick gallery: 126 td before signal → trade → 63 td after exit; white/black candles + green/red volume, Signal/Entry/Exit verticals, dotted entry/stop/target, MAE/MFE stats box.

PieceWhere
Rendererscripts/build_signal_charts.py — renders PNGs → charts/ (gitignored local mirror) and uploads to R2 under the charts/ prefix.
Shared logicscripts/signal_chart_common.py — stable key (chart_relpath) + MAE/MFE math, imported by BOTH the renderer and build_site so they can't drift.
Manifestbuild_site.py → dist/data/charts.json (path + MAE/MFE per trade; built in the non---no-mtm path, needs prices).
Servingfunctions/chartimg/[[path]].js streams PNGs from the CHARTS R2 binding (route /chartimg/* → R2 key charts/*). Binding declared in wrangler.toml.
Gallery UIsite/charts.html + site/assets/charts.js (+ .gallery/.viewer/.tradelist in style.css). Nav entry in common.js.
CI stepdeploy_site renders only NEW charts each run: build_signal_charts.py --all --upload --skip-existing (continue-on-error). R2 source of truth; PNGs are never bundled into the deploy.

Charts-specific failure modes

SymptomDiagnosis / fix
Gallery lists trades + captions, but every image is "Chart not generated yet"PNGs aren't reaching R2. Check the deploy log's "Render new signal charts → R2" step (it's continue-on-error, so it can fail quietly — e.g. mplfinance missing, R2 creds). Or the key changed. Fix: re-run the render (--all --upload --skip-existing locally, or redeploy).
ALL images fail (503), gallery list still fineCHARTS R2 binding didn't apply. Confirm wrangler.toml exists with pages_build_output_dir + the r2_buckets block, and the deploy log's Deploy step printed an R2 binding. Fallback: set it in the dashboard (Pages → seasonals-mslade → Settings → Functions → R2 bindings → CHARTS = seasonals-cache), then redeploy.
Charts tab says "No charts payload"charts.json wasn't built — build_site ran with --no-mtm (dev), or prices failed to load. meta.json → payloads.charts should be true.
Gotcha — config MUST be wrangler.toml, not .jsonc. cloudflare/wrangler-action@v3 installs wrangler 3.90.0, which ignores pages_build_output_dir in a .jsonc (deploy aborts: "Must specify a directory of assets… or configure pages_build_output_dir in your wrangler.toml"). The first chained deploy failed exactly this way. If you regenerate the config, keep it TOML. The deploy command is config-driven (pages deploy, no positional dir) so it relies on pages_build_output_dir.

Verify locally before deploying

Match the action's wrangler version (3.90.0), not your global 4.x — that's how the .jsonc bug was caught.

npx wrangler@3.90.0 r2 object put "seasonals-cache/charts/signals/52wh_Breakout/CRM_20231130.png" \
    --file charts/signals/52wh_Breakout/CRM_20231130.png --local
curl -I http://127.0.0.1:8788/chartimg/signals/52wh_Breakout/CRM_20231130.png   # -> 200 image/png

python scripts/build_signal_charts.py --all --upload

Cost: $0/mo — ~360 MB of PNGs sits inside R2's 10 GB free storage; reads/egress are free. Keys are stable (strategy+ticker+signal-date), so incremental runs only ever add the day's new trades.
